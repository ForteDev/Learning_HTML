HTML Images - HTML 이미지

이 웹페이지는 HTML 이미지를 공부하고 실습하기 위해 2020년 5월 18일에 작성한 페이지입니다. 주로 https://www.w3schools.com/html/html_images.asp에 근거한 내용을 담고 있습니다.

HTML 이미지 문법

이미지 태그는 <img>태그를 통해 정의합니다. 이미지 태그는 빈 요소입니다. 종료 태그를 갖지 않습니다.
src 속성을 통해서 이미지의 URL을 입력합니다.

alt - 대체 텍스트 속성


img 태그에 src 속성과 더불어 alt 속성을 자주 사용해요. alt 속성은 이미지의 대체 텍스트를 나타내요. 만일 이용자가 모종의 이유로 이미지를 보지 못할 경우(연결 속도가 느리거나, src 속성에 오류가 있거나, 맹인이라 스크린 리더를 사용해야 할 경우) 이미지 대신 대체 텍스트가 표시됩니다. 대체 텍스트가 이미지를 볼 수 없는 경우에 쓰이니 이미지를 설명하는 문구를 alt 속성에 적어야 하겠죠? 따라서 대체 텍스트는 선택이 아닌 필수입니다.

Width & Height - 이미지 크기 조정

이미지의 너비와 높이를 조정하는 두가지 방법이 있어요. 첫번째는 너비와 높이 조정을 위해 style 속성을 사용하는 방법입니다. 스타일 속성으로 너비와 높이를 설정하면 됩니다. 이미지의 픽셀을 정하거나 화면을 차지하는 비율(%)을 정하므로 크기를 조정할 수 있습니다.
이미지의 너비와 높이를 조정하는 두번째 방법은 너비와 높이 속성을 사용하는 것이에요. 너비와 높이 속성은 무조건 픽셀 단위로 정의됩니다.
두가지 방법중 어느 방법을 사용하든 간에 이미지의 너비와 높이를 명시하는 것이 좋아요. 너비와 높이를 정해놓지 않으면 이미지 로딩중에 웹페이지가 깜빡일 수 있어요.

width, height, style 속성중에 무엇을 사용해야 할까요? w3school에서는 style 속성을 사용하라고 추천해줍니다. 스타일 속성을 사용해야 웹페이지에 적용된 CSS가 이미지를 조정하는 것을 방지합니다.

아래 이미지는 너비 속성을 128 픽셀로 설정해놨습니다. 근데 CSS가 덮어써서 너비가 100%로 설정되었습니다.

아래 이미지는 스타일 속성을 사용했어요. 128 픽셀로 설정해놔서 이게 역으로 CSS를 덮어씁니다.

HTML Image Maps

HTML Image-maps를 사용해서 이미지 안에 클릭가능한 구역을 만들 수 있습니다. <map> 태그는 이미지 맵을 정의합니다. 이미지 맵을 사용하기 위해서는 이미지 태그에 usemap 속성을 추가하고 속성값에 #을 붙인다음 이미지 맵의 이름을 적어야합니다.
그 다음에 map 요소를 추가해줍시다. map 태그의 위치는 중요하지 않습니다. 이미지 태그 다음에 오지 않아도 됩니다. map 요소안에 name 속성을 추가하고 이미지 태그에서 정한 이름을 그대로 적어줍시다. 다음으로 이미지상에서 클릭할 수 있는 구역을 만들어 줍시다. <area> 요소로 구역을 만들어 줍시다. 구역 태그는 요소 태그안에 있어야 합니다. 구역의 형태와 좌표를 shape와 coords 요소에 적어 줍시다. 아 나머지는 설명하기 귀찮네요. 설명하려고 공부하는 게 아니니까 쇽쇽 넘어가 줍시다.

배경 이미지

HTML에서는 배경화면을 설정할 수 있습니다. 배경화면은 이미지 태그로 설정하진 않고 스타일 태그나 CSS에서 background-image 특성을 설정하여 적용합니다.

background-image: url(이미지 주소)

위와 같은 형태로 작성하자. 배경화면은 브라우저로 표시한 웹페이지 화면보다 작을 경우, 이미지를 반복해서 생성한다. 그럼 1도 안 이뻐지는데 이 기능을 방지하기 위하여 background-repeat: no-repeat 특성을 꼭 적어주자.
또한 배경 이미지를 설정한 요소 전체에 사진이 표시되길 바란다면 background-size: cover; 특성을 적어주고, 이미지가 제멋대로 확대하고 축소하는 현상을 방지하기 위해 background-attachment: fixed; 특성을 반드시 집어넣자. 그러면 스크롤을 움직여 웹페이지 이곳저곳을 활보해도 동일한 이미지가 표시된다. 스크롤한다고 이미지가 움직이지 않는다. 집어넣지 않으면 화면을 오므렸다 폈다하면서 이미지가 춤을 추게 된다. 다만, 이렇게 표시하면 이미지가 온전히 표시되지 못하고 요소가 표시된 구역 경계에서 짤리게 된다. 온전히 표시하고 싶다면 background-size: 100% 100%로 설정하자. 그럼 이미지가 요소 구역 크기에 딱맞게 확대된다. 그러나 모양이 하나도 이쁘지 않다. 그리고 웹페이지 전체에 걸쳐서 배경이미지를 나타내고 싶다면 body 태그에 배경화면을 입히면 된다. 그럼 기본 배경화면이 설정된다. body 태그 안에 있는 다른 요소에 또다른 배경화면을 입히면 어떻게 될까? 지금 보는 것처럼 body태그의 배경화면을 덮어써서 새로운 배경화면을 띄운다. 근데 문제는 이 웹페이지처럼 요소의 끝 = 웹페이지의 끝인 경우 화면 바닥부분에 body 태그에 입힌 배경화면의 발가락이 빼꼼하고 튀어나온다. 옥의 티네. 계속 공부하다보면 이 현상을 해결하는 방법도 깨닫게 되겠지.

배경 화면을 고정하고 전체 화면을 덮는게 보기에도 좋네. 지금 이 웹페이지처럼 말이야. 이 설정을 기억하면서 자주 활용하는 게 좋겠다.
